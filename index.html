Q1: Depth Testing

In Cube3D.cpp, we call glEnable(GL_DEPTH_TEST). Removing this call disables depth testing, causing OpenGL to draw triangles in the order they appear in the array. As a result, earlier faces (like red) may get overwritten by later ones, leading to deformation or flickering of the cube as it rotates.

Q2: Updating the Quit Condition

In the original program, the escape key (Escape) is used to exit the program. To also exit when the "Q" key is pressed, I modified the event handling code as follows:

if (windowEvent.type == SDL_EVENT_KEY_UP && (windowEvent.key.key == SDLK_ESCAPE || windowEvent.key.scancode == SDL_SCANCODE_Q)) {
    quit = true;
}

Q3: Light Direction Modification

In CubeLit.cpp, I changed the light direction vector in the fragment shader to point downward (e.g., vec3(0, 1, -1)) to simulate light coming from below the cube. The modified shader results in the cube being lit from below, giving it a different visual appearance.
